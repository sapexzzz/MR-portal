# Changelog 2.1.0

## Исправления

- **Клик по звезде избранного теперь одновременно выбирает метку**: При нажатии на звезду точка сразу становится текущей выбранной меткой, как если бы игрок нажал на саму строку для телепорта.
- **Убран лишний текст про избранное в правой панели GUI**: Сообщение о том, является ли метка избранной, больше не отображается — это уже достаточно понятно по звезде в списке.
- **Портал остаётся проходимым для других сущностей до реального закрытия**: Проверка положения владельца больше не блокирует проход через уже открытый портал для остальных сущностей и игроков. Пока портал визуально открыт, другие сущности могут входить в него до конца его жизни.

## Технические изменения

- `WaypointScreen.mouseClicked()` теперь сначала выбирает метку, а затем переключает избранное при клике по звезде.
- В `WaypointScreen.toggleFavorite()` добавлено обновление локального статуса выбора, чтобы GUI сразу вёл себя как после обычного выбора метки.
- В `PendingTeleportManager.tickInternal()` логика прохода владельца через портал отделена от логики прохода других сущностей: отсутствие владельца в исходном измерении больше не прерывает обработку остальных сущностей.
